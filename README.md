How can I ensure that my code, program, or software is functional and secure?

One of the most important things I learned in this course is that testing should be done throughout the entire development process. I used JUnit to test my Contact Service and made sure it met all of the project requirements. I also tested invalid inputs to verify that the program handled errors correctly. Running tests regularly helped me catch problems early and gave me confidence that my code was working as expected.

How do I interpret user needs and incorporate them into a program?

Understanding what the customer wants is the first step in building good software. Before writing any code, I carefully reviewed the project requirements to make sure I knew what features were expected. For this project, I included unique IDs, validation rules, and update functions because they were required by the client. Focusing on the user's needs helps create software that is reliable, easy to use, and meets the customer's expectations.

How do I approach designing software?

Before I begin coding, I like to create a plan for how the application will be organized. Breaking the project into smaller parts makes it easier to understand each class and method before writing the code. I also think about how I will test each feature while I'm designing it because writing tests early helps identify problems before they become bigger issues. Going forward, I plan to continue using this strategy because it helps me stay organized and build better software.
